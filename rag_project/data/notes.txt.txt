RAG stands for Retrieval Augmented Generation. It improves the accuracy of large language models by connecting them with external knowledge sources. The model retrieves relevant data first. The model then generates an answer using that retrieved information.

This method solves a common problem in language models. Models generate answers from training data only. Training data stays fixed after training. If the question requires new or private data, the model lacks access. RAG solves this problem by retrieving information from external documents during runtime.

Core idea
RAG combines two systems.

Retrieval system
Generation system

The retrieval system searches a knowledge base. The generation system uses the retrieved content to produce an answer.

How RAG works step by step

Data collection
You collect documents. Examples include PDF files, text files, website data, or company documents.

Example sources
Product manuals
Research papers
Company policies
Restaurant menus
Customer FAQs

Document preprocessing
Large documents get split into smaller pieces. These pieces are called chunks.

Example
A 50 page PDF gets split into 500 chunks.

Reason
Small chunks improve search accuracy.

Text embedding
Each chunk converts into a vector representation. A vector represents meaning in numbers.

Example embedding model
sentence transformers
all MiniLM L6 v2

Example vector
[0.12, 0.84, 0.44, 0.91 ...]

Chunks with similar meaning produce similar vectors.

Vector database storage
Vectors store in a vector database.

Common vector databases
Chroma
FAISS
Pinecone
Weaviate

Database stores two things
vector embedding
original text chunk

User query
A user asks a question.

Example
What is the refund policy for catering orders.

Query embedding
The system converts the question into a vector using the same embedding model.
Similarity search
The system searches the vector database.
It finds chunks with similar meaning to the query.

Example result
Top 3 relevant document chunks.

Context creation
The retrieved chunks combine with the question.

Example prompt

Question
What is the refund policy.

Context
Refund requests must be submitted within 48 hours.
Catering deposits remain non refundable.

Generation step
The prompt sends to an LLM.

Example models
Llama 3
Mistral
GPT models
Claude

The LLM reads the retrieved context and generates the answer.

Advantages of RAG

Up to date knowledge
Model accesses new data without retraining.

Private knowledge access
Use company documents securely.

Reduced hallucination
Answers rely on retrieved data.

Lower cost
No need to train a new large model.
